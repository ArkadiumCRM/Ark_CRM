# HR · Academy · M4 Modell + 16 Teilprozesse

Arkadium's Prozess-Framework für Executive-Search-Geschäftsabwicklung. Basiert auf 4 Hauptprozessen × 4 Teilprozessen = 16 Standardschritte. Eingeführt 30.07.2013, seither stabil.

**Vokabular:** Das M4-Modell matcht direkt die CRM-Prozess-Stages aus `ARK_STAMMDATEN_EXPORT_v1_3.md` §13 `dim_process_stages` und die Commission-Engine-Halbpunkt-Systematik.

## Die 4 Hauptprozesse

| Phase | Code | Name | Fokus | Haupt-Rolle |
|-------|------|------|-------|-------------|
| 1 | M1 | **MEET** | Kandidat kennenlernen · Matching zu Positionen | Research + Candidate Manager |
| 2 | M2 | **MATCH** | Kandidat bei Positionen/Unternehmen vorstellen · Pipeline-Begleitung | Candidate Manager + Account Manager |
| 3 | M3 | **MARKET** | Markt-Akquise · neue Mandate · Kunden-Kontakt | Account Manager |
| 4 | M4 | **MONEY** | Interview-Orga · Offer-Verhandlung · Placement · Abschluss | Candidate Manager + Account Manager |

**Ziele des Modells:**
1. Aufteilung Verantwortlichkeiten auf mehrere MA (CH = Kandidat / TL = Kunde)
2. Standardisierung · Schulung · Wissensvermittlung aller Prozessschritte
3. Messung Fortschritt via KPIs pro Schritt
4. Prozess-Optimierung

## Die 16 Teilprozesse

### M1 · MEET

| Code | Modul | Kurz-Beschreibung | Rolle |
|------|-------|-------------------|-------|
| 1.1 | **Ident** | Identifikation interessanter Kandidaten in XING/LinkedIn · Long-List-Bau | Research |
| 1.2 | **Hunt** | Telefonische Erstansprache der identifizierten Kandidaten | Research + CM |
| 1.3 | **CV Chase & Briefing Date / Q&D Briefing** | Fehlende CVs/Briefing-Dates nachtelefonieren · Q&D = Quick-&-Dirty-Briefing | CM |
| 1.4 | **Briefing / Doc Date** | Tiefes Gespräch mit Kandidaten: Kompetenzen · Präferenzen · Motivation | CM |

### M2 · MATCH

| Code | Modul | Kurz-Beschreibung | Rolle |
|------|-------|-------------------|-------|
| 2.1 | **Doc Chase** | Fehlende Arbeitszeugnisse/Diplome nachtelefonieren | CM |
| 2.2 | **Pre-/ReLeading** | Welche aktuellen Positionen + Unternehmen passen zum Kandidaten? | CM |
| 2.3 | **Pre-/ReLeads Presentation** | Marktübersicht dem Kandidaten präsentieren | CM |
| 2.4 | **GO's Fishing** | GOs für Stellen abholen, die Kandidat zunächst absagte | CM |

### M3 · MARKET

| Code | Modul | Kurz-Beschreibung | Rolle |
|------|-------|-------------------|-------|
| 3.1 | **Docs Generating** | Dossier in standardisierter Reihenfolge aufbereiten (CV · Abstract · Unterlagen) | CM |
| 3.2 | **Abstract** | Kurze + knackige Kandidaten-Zusammenfassung schreiben (Template aus [[dok-generator]]) | CM |
| 3.3 | **Akquise / Home Run Call** | Kontaktierung Kunde · Kurz-Besprechung Profil + heikler Punkte | AM |
| 3.4 | **CV sent** | Dossier · Abstract · AGB per Email an Kunde | AM |

### M4 · MONEY

| Code | Modul | Kurz-Beschreibung | Rolle |
|------|-------|-------------------|-------|
| 4.1 | **TI / 1st Orga / VT Coaching & Debriefing** | Organisation 1. Interview (TI = Telefon-/Teams-Interview) · Coaching Kandidat vor · Debriefing Kandidat + Kunde danach | CM (Debrief) + AM |
| 4.2 | **2nd Orga / VT Coaching & Debriefing** | Organisation 2. Interview · Coaching · Debriefing | CM + AM |
| 4.3 | **Offer** | Vertragsofferte in Erfahrung bringen · Abwägen · Verhandeln · Kandidat zum Abschluss führen | AM (+ CM emotional support) |
| 4.4 | **Placement** | Abschluss · Garantie-Prozess starten · Archivieren | AM |

## Academy-Modul-Metadaten (Schema §14.4-Erweiterung)

Alle 16 Teilprozess-Module werden als separate Academy-Module in `dim_academy_modules` geseedet. Typisch:
- Dauer: 2–3 h pro Modul
- Format: Schulungs-PDF + Praxis-Pair-Work + Role-Play
- Reihenfolge: strict · M1 → M2 → M3 → M4
- Mandatory: Consultant für alle 16 · Research-Analyst typisch nur M1.1–M1.3 + Grundlagen M4 (Placement-Terminologie)

**Zusätzliche Module aus PDFs (variations):**
- `M 3 3 Akquise & Home Run Call.pdf` (Standard)
- `M 3 3 Akquise Home Run Call mit Account Hunting.pdf` (Variante mit AM-Fokus)

### Trainer

Peter-Entscheidung §11C-3: Multiple Trainer je Modul. Typische Zuordnung:
- Communication Edge + M1 Grundlagen: Head-of-Department
- M2–M3: erfahrene Consultants (Team-Leader)
- M4 Placement: Head-of-Department + Founder (Praemium-Victoria-Kontext)

## Mapping zu Commission-Engine

**Halbpunkt-System:**
- M1–M2 (Kandidaten-Seite) → **Candidate Manager Halbpunkt (50%)**
- M3–M4 (Kunde-Seite) → **Account Manager Halbpunkt (50%)**
- M1.1 (Ident als Research) → Researcher-Pauschale (falls Researcher-Rolle beim Deal)
- §5.3-Praemium-Regel: Rollen-Wechsel Researcher → Consultant startet 3-Mt-Grace-Period für alte Ident/Hunt-Vermittlungen
